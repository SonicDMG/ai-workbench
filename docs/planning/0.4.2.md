# 0.4.2 release plan

Hardening continues on the 0.4.x line, with **one new feature**: store-level
keyset pagination for the chat surface. No wire-contract change
(`{ items, nextCursor }` is preserved), no data migration.

Organised as the A–E workstreams + a release step, mirroring
[`0.4.0.md`](0.4.0.md).

## A — Feature: store-level keyset pagination ✅

The SQLite control-plane driver (0.4.0) already fixed the *write*-side
quadratic file rewrite. The remaining gap was the *read* path: the chat list
routes fetched the whole conversation and sliced in memory. 0.4.2 pushes a
keyset cursor into every backend for messages **and** conversations.

- **A1 — primitive.** Additive keyset cursor codec in
  [`lib/pagination.ts`](../../runtimes/typescript/src/lib/pagination.ts):
  `KeysetKey`, `clampLimit`, `compareKeyset`, `isAfterKeysetCursor`,
  `encode/decodeKeysetCursor`, `paginateKeyset`. The existing offset
  `paginate()` is left untouched, so the bounded control-plane surfaces are
  unaffected (no shared-signature churn). Direction-aware (messages ASC,
  conversations DESC); the `id` tiebreaker is always ascending so cursors
  strictly advance even when rows collide at millisecond resolution.
  Oversized / malformed / legacy-offset cursors → `400 invalid_cursor`.
- **A2 — backends.** `listChatMessagesPage` + `listConversationsPage` on the
  control-plane store, returning `{ items, nextKey }`, alongside the untouched
  full-history `listChatMessages` / `listConversations`. A new `readPage` seam
  on `FileStoreState` (file reads the table + filters; SQLite range-scans the
  `pk` index). memory + Astra implement in their own slices. Every backend
  funnels its final slice through the shared `paginateKeyset`, so correctness
  lives in one tested place.
- **A3 — routes.** `routes/api-v1/agents.ts` lists via the store `*Page`
  methods + cursor codec. The messages route over-fetches and fills to `limit`
  *visible* rows (`pageVisibleMessages`), stamping the cursor from the last kept
  row — a page may be short/empty with a non-null cursor. Verified that
  `assemblePrompt` + `mcp/server.ts` still read full history (no truncation).
  `docs/api-spec.md` documents the keyset cursor semantics.

**Acceptance:** keyset contract tests across all four backends (ordering,
stability under churn, same-millisecond tiebreak, limit-independence); an
HTTP-level drain test through tool scaffolding; `400 invalid_cursor` route test.

## B — Reliability hardening ✅

Audit finding: the planned items were already shipped in 0.4.0 — the SSE helper
guarantees one terminal event + `Last-Event-ID` resume, the Astra fetcher
retries transient errors with a replayable-body guard, and `SIGTERM` already
drains HTTP / stops the sweeper + poller / closes the store / flushes OTel.

One genuine gap closed: a long-lived **job-events SSE stream** held its
connection open through the shutdown drain window. A `shutdownSignal` on
`ReadinessSignal`, threaded to `runJobEventStream`, now ends the stream on
shutdown (the client reconnects + resumes via `Last-Event-ID`); no fake
terminal `done` is emitted.

## C — Security hardening ✅

Re-audit finding: the SSRF guard is otherwise strong (`EndpointBaseUrlSchema`
blocks IMDS / link-local / unique-local + IPv6-mapped-IPv4; `safeFetch` forces
`redirect: "error"`; the MCP client reuses the validator).

One genuine, exploitable gap closed: the `native:fetch` tool (model-supplied
URL → reachable via prompt injection) range-checked only **literal** IPs.
`resolvedSsrfRejectReason` now resolves DNS names and validates every resolved
address, failing closed. Cursor input validation (the other C item) was
delivered in A1's codec; a route-level `400 invalid_cursor` test was added.

## D — Conformance & test coverage ✅

Audit finding: the chat-surface conformance gap the roadmap flagged was already
closed in 0.4.0's D4 (`chat-message-crud-basic`, `chat-message-stream`,
`chat-sse-tool-call`, `agent-crud-basic`). The drift guard confirms the A2/A3
route rewrites produce byte-identical fixtures. No pagination conformance
scenario was added — opaque keyset cursors are not stable across deploys and do
not fixture well.

Added a cross-backend limit-independence property test. Coverage verified above
the 80% bar (82.4% statements / 83.8% lines overall; the feature modules are
strong — `pagination.ts` 100% lines, the backend Page methods 92–97%).

## E — Cleanup & deps ✅

Conservative dependency refresh (patch/minor only) across all workspaces;
`npm audit` → **0 known vulnerabilities**. knip confirms 0.4.2 introduced no
dead code, and the pre-existing knip findings (20 files, ~205 lines of unused
exports / types / duplicate aliases, unrelated to chat pagination) were removed
in this release — `tsc`, `knip`, `biome`, and the runtime suite stay green.

## R — Release ✅ (stop before tag)

Versions bumped 0.4.1 → 0.4.2 (5 `package.json` + lockfiles + the two
`version.ts` constants); `CHANGELOG.md` + the in-app what's-new modal +
[`whats-new-0.4.2.md`](../whats-new-0.4.2.md) updated. The repo hand-maintains
the changelog (no per-slice changesets), so the entry is written by hand to
match 0.4.0/0.4.1. The `v0.4.2` tag — which fires the Docker + CLI release
workflow — is **pushed by a human**, not as part of this work.
